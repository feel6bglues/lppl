## 三、测试结果差异分析

### 问题1：阶段分布失衡

| 阶段 | 实际占比 | 预期占比 | 差异原因 |
|------|---------|---------|---------|
| markdown | 73.9% | 40-50% | 判定阈值过松 |
| unknown | 14.6% | 10-15% | 合理 |
| markup | 10.3% | 20-25% | 判定阈值过严 |
| accumulation | 0.7% | 5-10% | 判定阈值过严 |
| distribution | 0.5% | 5-10% | 判定阈值过严 |

**根因**: `_step1_phase_determine` 中 `short_trend_pct <= -0.03` 阈值过低

### 问题2：信号覆盖率极低

| 信号 | 实际占比 | 预期占比 | 差异原因 |
|------|---------|---------|---------|
| no_signal | 73.7% | 40-50% | 信号检测过严 |
| spring | 0.3% | 2-3% | Spring检测过严 |
| accumulation | 0.4% | 3-5% | 阶段判定过严 |

**根因**: `_step3_phase_c_t1` 中Spring检测条件过严

### 问题3：置信度矩阵失效

| 置信度 | 实际占比 | 预期占比 | 差异原因 |
|--------|---------|---------|---------|
| D | 82.5% | 40-50% | 条件过严 |
| C | 17.5% | 30-40% | 合理 |
| B | 0% | 15-20% | 条件过严 |
| A | 0% | 5-10% | 条件过严 |

**根因**: 5项条件几乎无法同时满足

### 问题4：Markdown阶段收益为正

| 指标 | Markdown | 预期 | 差异原因 |
|------|---------|------|---------|
| 平均收益 | 6.54% | <0% | 判定过松 |
| 超额收益 | 7.47% | <0% | 判定过松 |

**根因**: 大量股票被误判为Markdown

## 四、优化建议优先级

### P0: 立即修复

1. **调整Markdown判定阈值**
   - `short_trend_pct <= -0.03` → `short_trend_pct <= -0.05`
   - `current_price < ma20` → `current_price < ma20 * 0.95`

2. **添加Distribution阶段检查**
   - 在 `rule2_no_long_in_markdown` 中添加Distribution检查

3. **调整量能窗口**
   - `rolling(window=20)` → `rolling(window=30)`

### P1: 短期优化

4. **放宽Spring检测条件**
   - `row["low"] < low_bound` → `row["low"] < low_bound * 1.03`
   - `row["close"] >= low_bound * 0.99` → `row["close"] >= low_bound * 0.97`

5. **优化置信度矩阵**
   - A级: 4项满足（放宽BC定位条件）
   - B级: 3项满足

6. **添加跳空缺口检测**
   - 在 `_step2_effort_result` 中添加缺口检测

### P2: 中期优化

7. **完善Step 5字段填充**
   - 多周期一致性声明
   - 执行前提
   - 5项置信度核对

8. **添加多种目标位来源**
   - 大阴线起跌点
   - 缺口下沿

9. **添加涨跌停流动性警告**
   - 在止损计算中添加流动性风险警告
