> 生成日期: 2026-05-13
> 数据源: 通达信日线 (TDX .day)
> 股票池: 5,199只A股 (stock_list.csv)
> 研究周期: 三轮迭代 (2016-2026 / 2018-2026 / 2020-2025) + 最终双策略验证 (2020-2026)

## 一、研究背景

### 1.1 起点: 三策略组合

原始方案基于 `deployment_plan.md` 的三策略等权组合:

| 策略 | 权重 | 说明 |
|------|------|------|
| Wyckoff v2+P3 | 0.34 | 威科夫量价结构分析 |
| MA5/20 金叉 | 0.33 | 均线趋势跟踪 |
| 短期反转 | 0.33 | 超跌反弹 (5日跌超-5%触发) |

### 1.2 三期验证发现问题

通过对三个独立时间窗口的交叉验证，发现短期反转策略持续退化:

| 时间段 | Wyckoff 夏普 | MA5/20 夏普 | 短期反转夏普 | 组合夏普 |
|--------|:----------:|:----------:|:----------:|:------:|
| 2016-2026 (20窗) | 0.395 | 0.662 | **1.187** | 0.974 |
| 2018-2026 (20窗) | 1.059 | 0.185 | **0.754** | 0.838 |
| 2020-2025 (20窗) | 2.133 | 1.695 | **-0.295** | 1.597 |

**关键发现**: 短期反转在2020-2025年已变为负期望策略 (夏普-0.295, 蒙特卡洛夏普>0概率0%)，原因:
- A股市场风格从"散户追涨杀跌"转向"机构主导的趋势延续"
- 2016-2025十年间，A股反转效应逐步减弱，动量效应增强
- 该策略信号覆盖虽高 (49.6% 交易占比) 但已成为组合拖累项

### 1.3 决策: 移除短期反转

基于三期验证数据，决定从组合中移除短期反转，保留 Wyckoff + MA5/20 构成双策略组合。

## 二、策略详细设计

### 2.1 策略架构

```
输入: 个股OHLCV日线数据 (通达信)
  │
  ▼
┌─────────────────────────────────────┐
│ Step 0: 市场制度过滤                 │
│   沪深300 MA120/MA60 判断牛/熊/盘整  │
│   熊市 → 仅Wyckoff可交易, MA禁用    │
└─────────────────────────────────────┘
  │
  ├──▶ Wyckoff v2+P3 (权重0.50)
  │     ├ 400天日线 + 120周周线 + 40月月线
  │     ├ 多周期结构分析
  │     ├ 信号→入场/止损/目标→盈亏比判定
  │     └ 退出: 止盈50%+追踪止损 / 时间止损 / 最大持有
  │
  └──▶ MA5/20 金叉 (权重0.50)
        ├ 5日均线上穿20日均线
        ├ 持有至死叉出现 (最长120天)
        └ 无止损/止盈, 纯趋势跟踪

输出: 交易信号 (买卖点)
```

### 2.2 Wyckoff v2+P3 策略

#### 核心逻辑

基于 WyckoffEngine (lookback_days=400, weekly_lookback=120, monthly_lookback=40) 进行多周期结构分析:

1. **Step 0**: BC/TR 定位扫描 — 识别买入高潮/交易区间边界
2. **Step 1**: 大局观定调 — 确定威科夫四大相位 (Accumulation/Markup/Distribution/Markdown)
3. **Step 2**: 努力与结果 — 量价博弈特征分析
4. **Step 3**: Spring/UTAD 检测 + T+1 风险评估
5. **Step 3.5**: 反事实压力测试
6. **Step 4**: 盈亏比投影 (止损=关键低点×0.995, RR≥2.5)
7. **Step 5**: 交易计划输出

#### 信号触发条件

- 方向非"空仓观望"
- 市场制度非熊市 (bear regime 禁止)
- 至少100天历史数据
- 结构分析通过: 有明确 BC/TR 边界 + 盈亏比达标

#### 退出逻辑 (持仓管理)

| 退出模式 | 触发条件 |
|---------|---------|
| 止损 (stop_loss) | 价格触及关键结构低点×0.93 |
| 目标止盈 (target_50pct) | 价格上涨至目标价位后追踪止损 |
| 时间止损 (time_stop) | 持仓超过制度设定时间阈值 (bull:60d/range:45d/bear:90d) |
| 追踪止损 (trailing_stop) | 止盈后回撤超过 ATR×atr_mult |
| 最大持有 (max_hold) | 达到制度最大持有天数 (bull:120d/range:90d/bear:180d) |

#### 制度参数

| 市场制度 | ATR乘数 | 时间止损(天) | 最大持有(天) |
|---------|:-------:|:----------:|:----------:|
| Bull | 3.0 | 60 | 120 |
| Range | 1.5 | 45 | 90 |
| Bear | 2.5 | 90 | 180 |
| Unknown | 2.0 | 60 | 120 |

### 2.3 MA5/20 金叉策略

#### 核心逻辑

纯趋势跟踪策略, 无市场制度过滤, 无止损/止盈:

**入场条件**:
- 前一日: MA5 ≤ MA20 (无金叉)
- 当日: MA5 > MA20 (金叉触发)

**退出条件**:
- MA5 < MA20 (死叉出现)
- 最长持有120天

**计算方法**:
```python
ma_fast = SMA(close, 5)    # 5日均线
ma_slow = SMA(close, 20)   # 20日均线
# 入场: prev_ma5 <= prev_ma20 AND curr_ma5 > curr_ma20
# 退出: curr_ma5 < curr_ma20
```

#### 特点

- 纯规则驱动, 无参数拟合
- 低频交易, 年化约8-12次
- 在趋势市中表现优异, 震荡市中磨损

## 十、代码合规审查报告

> 审查日期: 2026-05-13
> 审查范围: `scripts/run_dual_strat_wyckoff_ma.py`, `src/data/tdx_loader.py`, `src/data/tdx_reader.py`, `src/wyckoff/engine.py`
> 审查项目: T+1合规 / 交易成本 / 前复权数据

### 10.1 T+1 合规审查

**结论: ✅ 基本合规，但有边角问题**

#### Wyckoff 策略

入场机制:
```python
entry = we if use_we else cc        # 信号日收盘价入场
f = df[df["date"] > a]               # 未来数据从次日(D+1)开始
for _, rw in f.iterrows():           # 逐日检查止损/止盈
    if lo <= ss: break               # 最早D+1日可触发止损
```

- ✅ 入场日为 D（信号日收盘价）
- ✅ 止损检查从 D+1 日开始（满足T+1，前一交易日买入的股票今日可卖）
- ✅ 止盈检查从 D+1 日开始
- ⚠️ 边角问题: 若信号日盘中已触发止损，模型仍按收盘价买入（无法避免，EOD回测的固有限制）

#### MA5/20 金叉策略

入场机制:
```python
entry = float(h.iloc[-1]["close"])   # 信号日收盘价入场
fut = df[df["date"] > a]              # 未来数据从次日开始
for i in range(5, min(120, len(fut)-20)):
    ...  # 检查死叉
```

- ✅ 入场日为 D（信号日收盘价）
- ✅ 退出最早在 D+5 日以后（MA5 < MA20 需要时间确认）
- ✅ 纯多头持有，无T+1冲突

#### 短期反转策略 (已移除)

原策略有T+1风险:
```
原逻辑: 当日收盘买入，持有5日
问题: 无止损保护，若D+1日跌停则无法卖出
解决: 已从组合中移除
```

### 10.2 交易成本审查

**结论: ❌ 严重缺失 — 所有策略均未扣除交易成本**

#### 现状

`run_dual_strat_wyckoff_ma.py` 中收益计算:
```python
# Wyckoff
tr = (ep - entry) / entry * 100    # 无成本扣除

# MA5/20
tr = (ep - entry) / entry * 100    # 无成本扣除
```

A股实际交易成本应包含:

| 项目 | 费率 | 方向 | 说明 |
|------|:----:|:----:|------|
| 佣金 | 万2.5 (0.025%) | 双边 | 券商佣金最低收费 |
| 印花税 | 千1 (0.100%) | 卖出单边 | 国家收取 |
| 滑点 | 万5 (0.050%) | 双边 | 买卖价差+冲击成本 |
| **买入成本** | **0.075%** | 买入 | 佣金+滑点 |
| **卖出成本** | **0.175%** | 卖出 | 印花税+佣金+滑点 |
| **完整回合** | **0.250%** | 双边 | 一次买入+一次卖出 |

#### 成本影响测算

| 单笔收益 | 成本侵蚀 | 净收益 | 侵蚀占比 |
|:-------:|:-------:|:-----:|:-------:|
| +0.3% | -0.25% | +0.05% | **83%** |
| +0.5% | -0.25% | +0.25% | **50%** |
| +1.0% | -0.25% | +0.75% | 25% |
| +2.0% | -0.25% | +1.75% | 12% |
| +5.0% | -0.25% | +4.75% | 5% |
| +7.0% | -0.25% | +6.75% | 4% |

**低收益策略受冲击最大**: 短期反转平均收益仅+0.49%，成本侵蚀50%+
Wyckoff平均收益+5.02%，成本侵蚀5%，影响较小

#### 修复方案

在双策略脚本的收益计算中添加成本模型:

```python
# 在 trade_wyckoff 和 trade_ma 中添加
COST_BUY = 0.00075   # 买入成本
COST_SELL = 0.00175  # 卖出成本

# 收益计算改为
tr = ((ep - entry) / entry - COST_BUY - COST_SELL) * 100
```

#### 对策略的影响

**对MA5/20金叉策略的直接影响**:

不复权数据在除权除息日产生价格跳空，导致:

```
场景: 工商银行 2024-07-16 除权

MA5计算:
  除权前: MA5=(6.200+6.080+5.930+6.000+5.940)/5 = 6.030
  除权后: MA5=(5.890+6.200+6.080+5.930+6.000)/5 = 6.020
  一跳之间: MA5从6.030骤降至6.020

MA20计算:
  20日均线包含了12个除权前的价格和8个除权后的价格
  跳空导致MA20与MA5的关系被扭曲

金叉/死叉误触发:
  除权前: price=6.200, MA5=6.030, MA20≈5.900 → 正常金叉状态
  除权日: price=5.890, MA5=6.020, MA20≈5.910
    → MA5相对价格突然升高 → 可能维持金叉状态
  除权后3-5天: 价格回升至6.000附近
    → 若MA5下穿MA20 → 假死叉信号
```

**对Wyckoff策略的间接影响**:
- TR边界（交易区间上下沿）在除权后被错误定位
- 支撑位/阻力位因价格跳空而偏移
- Spring信号（价格刺穿下边界）可能因跳空被误判
