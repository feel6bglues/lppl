# 融合测试方案：渐进式动量增强实用指南

日期：2026-03-31
版本：v1.0

**核心理念**：先稳住基础（MA+ATR），再逐步引入动量因子，每阶段验证通过后再进入下一阶段。

---

## 1. 方案融合思路

### 1.1 两个方案的优点提炼

| 来源方案 | 核心优点 | 为什么重要 |
|---------|---------|-----------|
| **新手指南** | 执行流程清晰（数据→指标→信号→回测→报告） | 保证可复制性，减少执行错误 |
| **新手指南** | LPPL只做风控不做买入 | 避免过度优化，保持系统稳健 |
| **新手指南** | 阶梯仓位管理（0/0.33/0.66/1.0） | 风险可控，交易平滑 |
| **新手指南** | 参数调优顺序（趋势→波动→风控） | 避免同时调多参数导致混乱 |
| **动量方案** | 动态波动率缩放（Harvey et al.） | 高波动自动降仓，防范回撤 |
| **动量方案** | 价格动量因子（Jegadeesh & Titman） | 核心alpha来源，提升收益 |
| **动量方案** | 市场状态分层（趋势/震荡/崩盘/复苏） | 适应性策略，不同市况不同应对 |
| **动量方案** | 52周高点效应（George & Hwang） | 心理锚点，增强突破信号质量 |
| **动量方案** | 学术支撑扎实 | 降低试错成本，有据可依 |

### 1.2 融合原则

```
融合的两大原则：
┌─────────────────────────────────────────────┐
│ 1. 保留原则：新手指南的执行框架和控制逻辑全部保留        │
│ 2. 叠加原则：动量因子作为增强层叠加，每个因子有独立开关   │
└─────────────────────────────────────────────┘

融合后的架构：
┌─────────────────────────────────────────┐
│  Layer 4: 风控层 (LPPL状态机) - 新手指南   │
│  Layer 3: 波动层 (ATR + 动态缩放) - 融合   │
│  Layer 2: 动量层 (价格动量 + 52W高点) - 新增 │
│  Layer 1: 趋势层 (MA交叉) - 新手指南       │
│  Layer 0: 数据层 - 复用既有               │
└─────────────────────────────────────────┘
```

### 阶段概述

| 阶段 | 名称 | 核心改造 | 修改文件数 | 预期工期 | 回退成本 |

## 阶段 1：波动增强（低风险高收益）

### 核心改造
在既有ATR基础上，增加**动态波动率缩放**（Harvey et al. 2018）。

**为什么先做这个？**
- 风险最低：只改仓位大小，不改交易方向
- 效果最好：高波动期自动降仓，直接降低最大回撤
- 实现简单：复用已有ATR计算，只加一行缩放公式

### 具体改造

```python
# src/investment/backtest.py 修改 _compute_indicators

def _compute_indicators(df, signal_config):
    enriched = df.copy()

    # ===== 现有代码（保留）=====
    enriched["ma_fast"] = enriched["close"].rolling(...)
    enriched["atr"] = ...
    enriched["atr_ratio"] = ...

    # ===== 新增：动态波动率缩放（阶段1）=====
    if signal_config.enable_volatility_scaling:
        # 计算已实现波动率（21日年化）
        enriched["returns"] = enriched["close"].pct_change()
        enriched["realized_vol_21"] = (
            enriched["returns"].rolling(21, min_periods=1).std() * np.sqrt(252)
        )

        # 动态仓位缩放因子：目标15% / 实际波动率，限制2倍
        target_vol = signal_config.target_volatility  # 默认0.15
        enriched["vol_scale_factor"] = (
            target_vol / enriched["realized_vol_21"].clip(lower=0.05)
        ).clip(upper=2.0)

    return enriched

# 在 evaluate 函数中使用
if signal_config.enable_volatility_scaling:
    # 仓位 = 目标仓位 × 波动率缩放因子 × ATR限制 × LPPL限制
    vol_cap = signal_config.full_position * row["vol_scale_factor"]
    position_cap = min(lppl_cap, atr_cap, vol_cap)
```

## 阶段 2：动量引入（核心增强）

### 核心改造
引入**20日/60日价格动量**作为第二层确认，与MA交叉形成**动量共振**。

**为什么放在第二阶段？**
- 需要阶段1的波动控制作为基础（防止动量失效期的高波动）
- 是提升收益的核心因子
- 需要新增计算逻辑，复杂度中等

### 具体改造

```python
# src/investment/backtest.py 修改 _compute_indicators

def _compute_indicators(df, signal_config):
    enriched = _compute_indicators_base(df, signal_config)

    # ===== 新增：动量因子计算（阶段2）=====
    if signal_config.enable_momentum_factor:
        # 多周期动量
        for window in signal_config.momentum_windows:  # [20, 60]
            enriched[f"momentum_{window}"] = (
                enriched["close"] / enriched["close"].shift(window) - 1
            )

        # 合成动量得分（简单平均）
        momentum_cols = [f"momentum_{w}" for w in signal_config.momentum_windows]
        enriched["momentum_score"] = enriched[momentum_cols].mean(axis=1)

        # 动量方向（正负）
        enriched["momentum_direction"] = np.sign(enriched["momentum_score"])

    return enriched

# 新增信号评估函数 _evaluate_momentum_ma_atr
def _evaluate_momentum_ma_atr(row, signal_config, current_position):
    """
    动量+MA+ATR 三层确认模型
    """
    # 第一层：MA趋势
    ma_bullish = row["ma_fast"] > row["ma_slow"]
    ma_bearish = row["ma_fast"] < row["ma_slow"]

    # 第二层：动量确认（关键增强）
    if signal_config.enable_momentum_factor:
        momentum_positive = row["momentum_score"] > 0
        momentum_strong = row["momentum_score"] > 0.05  # 5%以上算强动量

        # 共振条件：MA金叉 + 动量为正
        buy_signal = ma_bullish and momentum_positive
        # 增强条件：强势动量可提前入场
        early_buy = momentum_strong and row["ma_fast_slope"] > 0
    else:
        buy_signal = ma_bullish
        early_buy = False

    # 第三层：ATR波动率过滤（阶段1已加）
    if row["atr_ratio"] > signal_config.vol_breakout_mult:
        buy_signal = False  # 高波动禁止买入

    # 仓位计算（复用阶段1的波动缩放）
    ...

    return {"next_position": ..., "reason": ...}
```

## 阶段 3：状态分层（高级适配）

### 核心改造
增加**市场状态识别**（趋势/震荡/崩盘/复苏），不同状态调整动量策略权重。

**为什么放在第三阶段？**
- 需要阶段2的动量计算作为输入
- 复杂度较高，需要前两个阶段稳定运行
- 是优化阶段的最后一块拼图

### 具体改造

```python
# 新增市场状态识别函数

def detect_market_state(df, signal_config):
    """
    市场状态识别器
    """
    state = {}

    # 1. ADX趋势强度
    adx = calculate_adx(df, period=14)
    state["adx"] = adx

    # 2. 价格波动范围
    price_range = (df["close"] - df["close"].rolling(20).mean()).abs()
    state["in_range"] = price_range < df["close"] * 0.03  # 3%内算震荡

    # 3. LPPL状态（已有）
    lppl_phase = df["lppl_phase"]  # watch/warning/danger/none

    # 状态判定
    if lppl_phase == "danger":
        state["regime"] = "crashing"
    elif adx > 25 and not state["in_range"]:
        state["regime"] = "trending"
    elif adx < 20 and state["in_range"]:
        state["regime"] = "ranging"
    elif df["momentum_score"].iloc[-1] > 0.1 and adx > 20:
        state["regime"] = "recovering"
    else:
        state["regime"] = "neutral"

    return state
```

## 阶段 4：全面融合（最终优化）

### 核心改造
引入**52周高点效应**作为动量的补充锚点，完成四因子融合。

**为什么最后做？**
- 边际效益递减（前三个阶段已解决主要问题）
- 可作为独立验证（可与阶段3对比）
- 复杂度较高

### 具体改造

```python
# 在 _compute_indicators 中新增（阶段4）

if signal_config.enable_52w_high_factor:
    # 52周高点
    rolling_52w_high = enriched["close"].rolling(252, min_periods=1).max()
    enriched["proximity_52w_high"] = enriched["close"] / rolling_52w_high

    # 突破强度
    enriched["is_breakout"] = enriched["proximity_52w_high"] > 0.99
    enriched["is_near_high"] = enriched["proximity_52w_high"] > 0.95

    # 与动量共振：突破新高 + 动量正 = 最强信号
    enriched["breakout_momentum"] = (
        enriched["is_breakout"].astype(float) *
        np.maximum(enriched["momentum_score"], 0)
    )

# 买入条件融合（最终版）
def _evaluate_full_model(row, signal_config):
    # 因子得分
    trend_score = 1 if row["ma_fast"] > row["ma_slow"] else 0
    momentum_score = max(row["momentum_score"], 0)  # 只取正动量
    breakout_score = row.get("breakout_momentum", 0) * signal_config.breakout_weight

    # 综合得分（权重可调）
    composite_score = (
        trend_score * 0.3 +
        momentum_score * signal_config.momentum_weight +
        breakout_score * 0.25
    )

    # 波动率缩放（阶段1）
    vol_scale = row.get("vol_scale_factor", 1.0)

    # LPPL风控（新手指南）
    lppl_max_position = get_lppl_position_cap(row)

    # 最终仓位
    target_position = min(composite_score * vol_scale, lppl_max_position)

    return target_position
```

### 验证标准

| 检查项 | 通过标准 |
|--------|---------|
| 综合提升 | eligible ≥ 5/8，平均超额 ≥ 5%，平均回撤 ≤ -20% |
| 各因子贡献可区分 | 能单独关闭任一因子并看到差异 |
| 全周期稳健 | 在2018、2022、2024市场下跌期都有保护效果 |
| 参数稳定性 | 参数微调（±10%）不改变ranking顺序 |

## 5. 风险与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|---------|
| 波动缩放过度导致交易过少 | 中 | 收益下降 | 放宽target_volatility（0.15→0.18） |
| 动量因子失效（ momentum crash） | 中 | 回撤扩大 | 阶段3状态分层提供保护 |
| 状态切换过于频繁 | 低 | 交易成本高 | 增加状态确认天数（1→3天） |
| 参数过拟合 | 中 | 样本外失效 | 每阶段用2020-2024训练，2025-2026测试 |
| 新手指南执行框架被破坏 | 低 | 无法回退 | 每阶段保留独立开关，可随时关闭 |
