## 1. 这套策略到底在做什么

当前推荐的策略不是单一 LPPL，也不是单一均线，而是三层结构：

1. `MA20 / MA60` 负责主买卖方向
2. `ATR` 负责波动确认和仓位控制
3. `LPPL` 负责顶部疯狂状态机

一句话概括：

> 趋势决定能不能做，波动决定做多少，LPPL 决定什么时候必须收缩风险。

### 1.1 推荐的交易规则

买入端：

- `MA20` 上穿 `MA60` 时，允许买入
- `ATR` 处于低波动时，才允许加仓
- 价格重新站上短均线时，才确认不是假突破

卖出端：

- `MA20` 下穿 `MA60` 时，允许减仓
- `ATR` 进入高波动时，进一步压仓
- `LPPL danger <= 3 天` 时，允许清仓

LPPL 只做状态机，不做主买入逻辑。

### 1.2 你要记住的核心边界

1. `watch`：只观察，不改仓
2. `warning`：只允许减仓，不负责清仓
3. `danger`：只有距离危险期 3 天内，才允许清仓

这是这套策略的硬规则。

## 3. 新策略的代码设计顺序

新手不要一上来就改所有文件。  
正确顺序是：

1. 先把数据读出来
2. 再算指标
3. 再生成信号
4. 再跑回测
5. 再出报告
6. 最后做参数优化

### 3.1 第一步：数据读取

目标：

- 取出某个指数的日线数据
- 保证字段齐全：`date/open/high/low/close/volume`

建议直接复用：

```python
from src.data.manager import DataManager

dm = DataManager()
df = dm.get_data("000300.SH")
```

新手不要自己手写 CSV 读取逻辑。  
因为这个项目已经处理了本地数据、远程数据、缓存和校验。

### 3.2 第二步：计算指标

建议先算 4 组最基本的指标：

1. 均线
   - `MA20`
   - `MA60`
   - 可选：`MA120`

2. 趋势斜率
   - 看 `MA20` 最近几天是上升还是下降

3. 波动率
   - `ATR14`
   - `ATR14 / ATR_MA`

4. 回撤
   - 过去 N 天最大回撤
   - 当前价格相对滚动高点的跌幅

### 3.3 第三步：生成信号

建议把信号拆成两层：

#### 层 A：方向信号

- 金叉：允许买入
- 死叉：允许卖出

#### 层 B：过滤信号

- `ATR` 太高时，不能重仓买
- `LPPL warning` 时，不能继续加仓
- `LPPL danger <= 3 天` 时，允许清仓

### 3.4 第四步：回测执行

回测要回答的不是“有没有信号”，而是：

- 这笔信号最终有没有赚钱
- 回撤有没有失控
- 换手是不是太高

建议复用：

- [`run_strategy_backtest`](/home/james/Documents/Project/lppl/src/investment/backtest.py)
- [`summarize_strategy_performance`](/home/james/Documents/Project/lppl/src/investment/backtest.py)
- [`calculate_drawdown`](/home/james/Documents/Project/lppl/src/investment/backtest.py)

### 3.5 第五步：输出报告

回测结束后必须输出三类结果：

1. CSV
   - 每日信号
   - 每日净值
   - 交易流水
   - 汇总指标

2. 图表
   - 净值曲线
   - 回撤曲线
   - K 线叠加信号图

3. Markdown / HTML 报告
   - 让人能快速看懂结论

## 4. 具体怎么计算

这一节是新手最容易卡住的地方。  
不要背概念，直接看口径。

### 4.1 MA 交叉怎么判

最简单的判定方式：

- `MA20 > MA60`：趋势偏多
- `MA20 < MA60`：趋势偏空

如果要更稳一点：

- `MA20 > MA60 > MA120`：强多头
- `MA20 < MA60 < MA120`：强空头

### 4.2 ATR 怎么用

ATR 不是方向指标，是波动指标。

建议先算：

- `ATR14`
- `ATR_MA40`
- `atr_ratio = ATR14 / ATR_MA40`

解释：

- `atr_ratio <= 1.05`：低波动，适合买入
- `1.05 ~ 1.15`：中波动，只观察
- `>= 1.15`：高波动，减仓或禁止加仓

### 4.3 LPPL 怎么用

LPPL 在这套方案里只做状态机，不直接决定买入。

状态建议：

- `watch`
  - 只观察
  - 不改仓

- `warning`
  - 只允许减仓
  - 不负责清仓

- `danger`
  - 如果距离危险期大于 3 天：继续减仓
  - 如果距离危险期小于等于 3 天：清仓

### 4.4 仓位怎么走

建议用阶梯仓位，而不是一口气满仓或空仓：

- `0.0`
- `0.33`
- `0.66`
- `1.0`

常见动作：

- `buy`：`0.0 -> 0.33`
- `add`：`0.33 -> 0.66`
- `add`：`0.66 -> 1.0`
- `reduce`：反向降一级
- `sell`：在 `danger <= 3` 时直接清到 `0.0`

## 5. 怎么看结果

新手看结果时，不要只盯着收益。  
至少看 6 个指标。

### 5.1 必看指标

1. `annualized_return`
   - 年化收益

2. `annualized_excess_return`
   - 相对基准是否真的更好

3. `max_drawdown`
   - 最大回撤是否可接受

4. `trade_count`
   - 交易次数是否合理

5. `turnover_rate`
   - 换手是否过高

6. `whipsaw_rate`
   - 是否频繁来回打脸

### 5.2 怎么判断一组参数好不好

一组参数可以先看成“是否有资格继续优化”：

- 如果收益为正，但回撤太大，不能直接上线
- 如果回撤很小，但完全不赚钱，也不能上线
- 如果交易次数过少，说明信号太保守
- 如果换手和 whipsaw 很高，说明信号太噪

### 5.3 哪些结果值得保留

优先保留这类结果：

- 有正超额收益
- 最大回撤没有爆掉
- 交易次数不至于太少
- 换手率没有离谱

## 6. 怎么继续优化

优化不要一口气改十个参数。  
新手最容易把自己绕晕。

### 6.1 优化顺序

建议按这个顺序调：

1. 先调趋势参数
   - `MA20`
   - `MA60`
   - `MA120`

2. 再调波动参数
   - `ATR period`
   - `ATR MA window`
   - `buy_volatility_cap`
   - `vol_breakout_mult`

3. 最后调 LPPL 参数
   - `watch_days`
   - `warning_days`
   - `danger_days`
   - `danger_r2_offset`

### 6.2 为什么这个顺序重要

因为趋势和波动决定了大部分日常交易，LPPL 只是尾部风控。

如果你一开始就调 LPPL：

- 很容易把收益最好的那一段提前让掉
- 也很容易把优化目标带偏

### 6.3 调优时必须保持的原则

1. 每次只改一组参数
2. 每次只看一个问题
3. 每轮都保存结果
4. 不要把“探索评分”和“生产评分”混成一个

### 6.4 分指数优化

不要拿 8 个指数硬套一套参数。  
更合理的做法是按组优化：

- 大盘组：`000001.SH`、`000016.SH`、`000300.SH`
- 平衡组：`399001.SZ`、`000905.SH`
- 高弹性组：`399006.SZ`、`000852.SH`、`932000.SH`

不同组的交易节奏不一样，参数必须分组。

## 8. 新手最容易犯的错误

### 错误 1：重复造轮子

不要自己再写一套：

- 数据读取
- 回测引擎
- 报告生成
- 配置解析

这些项目里已经有了。

### 错误 2：把 LPPL 当成买点生成器

LPPL 最适合做风险状态机，不适合单独负责买入。

### 错误 3：一次调太多参数

每一轮只改一类参数。  
否则你不知道到底是 MA、ATR 还是 LPPL 起作用。

### 错误 4：只看收益，不看回撤

指数策略里，这通常是错的。  
收益高但回撤不可接受，仍然不适合上线。

### 错误 5：不保留实验记录

每次调优必须把：

- 参数
- 输出 CSV
- 结论
- 失败原因

都留下来。  
否则你下一轮会重复走老路。
