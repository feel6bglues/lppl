## 二、最佳实践研究

### 2.1 AlphaForge框架启示

根据AlphaForge论文(arXiv:2406.18394):
- 因子动态组合优于静态组合
- 需要因子有效性监控机制
- 建议使用加权评分而非简单投票

### 2.2 动态因子择时

根据Northern Trust研究:
- 因子表现具有周期性
- 需要根据市场状态调整因子权重
- 建议使用regime detection机制

### 2.3 交易频率控制最佳实践

根据量化社区经验:
- **冷却期**: 卖出后10-20天不交易
- **最小持仓**: 买入后至少持有5-10天
- **信号确认**: 需要2-3天确认
- **趋势过滤**: 只在趋势明确时交易

### 3.2 关键改进点

#### 改进1: 市场状态识别

```python
def detect_market_state(df, config):
    """识别市场状态: 趋势市 vs 震荡市"""
    # 趋势强度
    ma_fast = df['close'].rolling(20).mean()
    ma_slow = df['close'].rolling(60).mean()
    trend_strength = (ma_fast - ma_slow) / ma_slow
    
    # 波动率状态
    atr_ratio = df['atr'] / df['atr_ma']
    
    if abs(trend_strength) > 0.05 and atr_ratio < 1.1:
        return "trending"
    elif abs(trend_strength) < 0.02 and atr_ratio > 1.0:
        return "ranging"
    else:
        return "transition"
```

#### 改进2: 动态因子权重

```python
def get_factor_weights(market_state):
    """根据市场状态返回因子权重"""
    if market_state == "trending":
        return {"trend": 0.50, "volatility": 0.30, "bb": 0.20}
    elif market_state == "ranging":
        return {"trend": 0.20, "volatility": 0.30, "bb": 0.50}
    else:  # transition
        return {"trend": 0.35, "volatility": 0.35, "bb": 0.30}
```

#### 改进3: 强化交易抑制

```python
class TradeSuppression:
    def __init__(self, config):
        self.confirm_days = 3        # 信号确认天数
        self.cooldown_days = 15      # 卖出后冷却期
        self.min_hold_days = 10      # 最小持仓天数
        self.holding_days = 0        # 当前持仓天数
        self.cooldown_remaining = 0  # 剩余冷却天数
        
    def can_buy(self):
        return self.cooldown_remaining <= 0
    
    def can_sell(self):
        return self.holding_days >= self.min_hold_days
```
