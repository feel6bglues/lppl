# 威科夫多模态分析测试计划

**文档日期**: 2026-04-07  
**适用对象**: 测试工程师 / 实现工程师 / 评审人  
**文档目标**: 锁定测试范围、验收标准与发布阻断项

---

## 1. 测试目标

验证系统能够：

- 正确读取数据与图片
- 严格执行威科夫规则
- 正确处理 A 股风险约束
- 正确融合多模态证据
- 正确输出结构化工件与报告
- 在失败和冲突场景下保守降级

---

## 2. 单元测试

### 2.1 数据层

- symbol 标准化正确
- 指数与个股路径解析正确
- 文件输入校验正确
- 数据缺列时拒绝分析
- 数据行数不足时拒绝分析

### 2.2 规则引擎

- 找不到 BC 时直接 `D级 + abandon`
- Distribution / Markdown 不能输出多头
- Spring 后 T+3 冷冻期内只允许观察
- 反证更强时推翻多头结论
- `R:R < 1:2.5` 时强制放弃
- `no_trade_zone` 场景不给 trigger
- 成交量标签仅使用允许枚举

### 2.3 图像引擎

- 扫描图片格式正确
- symbol 归属规则正确
- timeframe 识别规则正确
- `unknown_tf` 回退正确
- 图像质量分级正确
- 模糊图像不能提升置信度
- 图像层不输出成交量绝对值

### 2.4 融合引擎

- 数据与图片冲突时保守降级
- 周线派发背景下日线多头降级
- 图片-only 模式不会给高置信交易计划
- 图片间互相冲突时整体降级

### 2.5 状态层

- Spring 状态正确写入
- 冷冻期日期正确生成
- 连续性追踪模板生成正确

---

## 3. 集成测试

### 3.1 数据-only 模式

验证：

- 可生成 JSON / CSV / MD / HTML / state
- 无图像输入时整链可运行

### 3.2 图片-only 模式

验证：

- 可生成 evidence / MD / HTML
- `confidence <= C`
- 不输出执行级计划

### 3.3 数据 + 图片融合模式

验证：

- 可生成全套工件
- 可输出冲突清单
- 可输出附录 A / B

---

## 4. 回归测试

必须验证：

- 不影响 `index_investment_analysis.py`
- 不影响 LPPL verify / walk-forward
- 不影响现有 reporting 测试
- 不影响现有 plot generator 测试

---

## 5. Smoke Test 建议

建议最小链路覆盖：

1. 单指数数据-only
2. 单个股数据-only
3. 单指数数据+图片
4. 单个股数据+图片
5. 图片-only

---

## 6. 验收标准

以下条件全部满足才可验收：

- 三种模式都能运行
- 强约束规则全部生效
- deterministic 报告可输出
- optional LLM 无配置时可自动回退
- 状态与 evidence 工件齐全

---

## 7. 发布阻断项

以下任一问题均阻断发布：

- 找不到 BC 但仍给方向性判断
- Distribution / Markdown 输出多头计划
- Spring 冷冻期内允许开仓
- 图像冲突时没有降级
- 图片-only 输出 A/B 级结论
- LLM 改写结构字段
- 缺少状态或 evidence 工件
