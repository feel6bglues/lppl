# A股威科夫多模态分析系统 PRD

**文档日期**: 2026-04-07  
**适用对象**: 产品负责人 / 策略负责人 / 工程负责人 / 测试负责人  
**文档目标**: 定义威科夫多模态分析系统的产品边界、目标、输入输出、成功标准与风险约束

---

## 1. 背景

当前仓库已经具备以下基础能力：

- 本地/缓存 OHLCV 数据读取
- 日线级 K 线绘图与 PNG 输出
- Markdown / HTML 报告输出
- 指数投资分析与 LPPL 验证链路

但系统仍缺少一条独立的、面向实战交易研究的威科夫分析链路，尤其缺少：

- 对 A 股特有交易制度的硬约束建模
- 对 K 线图片文件夹的批量读取与视觉结构分析
- 日线数据与多周期图片证据的交叉校验
- 可连续跟踪的结构化交易计划输出

---

## 2. 问题定义

现有系统只擅长数值序列分析，不具备以下能力：

1. 读取项目图表文件夹并对截图进行多周期结构研判
2. 结合日线 OHLCV 做“事实主判 + 视觉辅助”的交叉校验
3. 严格按威科夫流程输出可执行、可审计的 A 股交易计划
4. 在图像模糊、结构冲突、阶段不明时自动保守降级

本项目需要新增一条独立能力链路，解决上述问题。

---

## 3. 产品目标

新增一个独立 CLI 和完整输出链路，使系统能够：

- 读取 A 股指数或个股的 OHLCV 数据
- 读取项目图表文件夹中的一张或多张 K 线图片
- 识别多周期结构证据
- 将日线数据事实与图片证据融合
- 严格遵守威科夫与 A 股交易铁律
- 输出结构化结果、Markdown 报告、HTML 报告、状态文件与证据清单

最终目标不是“生成一篇像分析师写的文章”，而是：

> 产出一份由规则主导、证据可回放、结论可追踪、风险受约束的机构级交易分析结果。

---

## 4. 用户场景

### 4.1 场景一：仅数值数据分析

用户提供：

- `--symbol`
- 或标准 OHLCV 文件

系统完成：

- 日线威科夫结构分析
- A 股风险约束判断
- 报告输出

### 4.2 场景二：图表文件夹 + 数据交叉校验

用户提供：

- `--symbol`
- `--chart-dir`

系统完成：

- 读取项目已有 PNG/JPG 图表
- 识别周线 / 日线 / 60m 等多周期证据
- 与数据引擎交叉校验
- 输出统一结论

### 4.3 场景三：图片-only 视觉巡检

用户只提供图表文件夹。

系统允许：

- 生成视觉证据报告
- 生成低置信观察结论

系统不允许：

- 生成高置信执行级交易计划

---

## 5. 产品范围

### 5.1 In Scope

- A 股指数与沪深个股
- 日线 OHLCV 数据读取
- 项目图表文件夹图片扫描
- 多周期视觉证据提取
- 威科夫规则引擎
- 数据与图片融合引擎
- 结构化输出与连续性状态落盘
- Markdown / HTML 报告
- 可选 LLM 叙述增强

### 5.2 Out of Scope

- 实盘下单
- 自动仓位执行
- ETF / 港股 / 美股 / 期货 / 加密资产支持
- 精确 OCR 还原 OHLC 数值
- 分钟级行情源接入
- 订单流 / Level2 / 盘口分析

---

## 6. 核心原则

### 6.1 规则主判原则

日线结论必须由规则引擎主导，不能由 LLM 直接生成。

### 6.2 多模态交叉校验原则

数据用于精确事实，图片用于补充日线外多周期结构。

### 6.3 A 股生存原则

- 绝不输出做空建议
- 派发和 Markdown 场景只能输出空仓相关结论
- 必须执行 T+1 风险压力测试

### 6.4 保守优先原则

证据冲突、图像模糊、BC 不清、结构混乱时，一律降级。

### 6.5 可回放原则

所有关键结论必须能回溯到结构化字段或视觉证据。

---

## 7. 输入模式

系统支持三类输入：

1. 数据-only
   - `--symbol`
   - 或 `--input-file`

2. 图片-only
   - `--chart-dir`
   - 或 `--chart-files`

3. 数据 + 图片融合
   - `--symbol` / `--input-file`
   - `--chart-dir` / `--chart-files`

---

## 8. 输出模式

每次分析默认生成以下工件：

- `raw/analysis_<symbol>.json`
- `raw/image_evidence_<symbol>.json`
- `summary/analysis_summary_<symbol>.csv`
- `plots/<symbol>_wyckoff_overview.png`
- `plots/<symbol>_multiframe_evidence.png`
- `reports/<symbol>_wyckoff_report.md`
- `reports/<symbol>_wyckoff_report.html`
- `state/<symbol>_wyckoff_state.json`
- `evidence/<symbol>_chart_manifest.json`
- `evidence/<symbol>_conflicts.json`

---

## 9. 成功标准

### 9.1 功能成功标准

- 可读取指数/个股 OHLCV
- 可扫描图表文件夹中的图片
- 可识别图片时间周期
- 可输出完整 Step 0 ~ Step 5 报告
- 可生成结构化 JSON / CSV / 状态文件

### 9.2 分析成功标准

- 找不到 BC 时直接放弃
- Distribution / Markdown 严禁给多头执行建议
- Spring 出现后进入 T+3 冷冻期
- R:R 不足 1:2.5 时直接放弃
- 图像模糊时不能提高置信度

### 9.3 工程成功标准

- 不影响现有 LPPL 与 investment 主链路
- 单元 / 集成 / 回归测试通过
- 无 LLM 配置时仍可完整运行

---

## 10. 非目标

本期不追求：

- 图像精准数值复原
- 泛化到任意市场
- 替代人工复盘
- 自动实盘交易

---

## 11. 风险与约束

### 11.1 主要风险

- 图像质量不稳定
- 多周期图像命名不规范
- 数据与图片结论冲突
- 个股数据断层、停牌、除权处理影响结构判断

### 11.2 风险控制

- 图像证据分级
- 冲突矩阵保守裁决
- 数据-only 可独立运行
- LLM 仅做叙述增强

---

## 12. 发布边界

v1 交付边界定义为：

- 支持 A 股指数与个股
- 支持项目图表文件夹扫描
- 支持日线规则引擎
- 支持图片辅助多周期判断
- 支持融合与状态持久化
- 支持 deterministic 报告
- 支持 optional LLM 增强

只要以上能力全部可运行并通过测试，即可视为首个可交付版本。
