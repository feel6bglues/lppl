4. 标记已经失真的文档或补充勘误

**验收标准**:

- `docs/README.md` 恢复成纯索引
- 任一结论都能追溯到具体 `output/*csv` 或 `reports/*.md`

### 任务 B：固化当前基线

**目标**:

- 明确当前唯一官方基线是 `ma_atr_turnover_gap_fix`

**执行项**:

1. 汇总 `IS/OOS/FULL` 的核心指标到一张基线表
2. 明确 `399006.SZ` 是唯一例外样本
3. 后续所有新策略都必须对比该基线

**验收标准**:

- 后续报告均包含“相对当前基线”的对照项

### 任务 C：启动长持仓策略样本内测试

**目标**:

- 先验证长持仓策略有没有基本价值

**执行顺序**:

1. 跑 `2012-01-01 ~ 2020-12-31` 样本内测试
2. 仅跑大盘组 `000001.SH,000016.SH,000300.SH,000905.SH`
3. 记录每个组合的：
   - `annualized_excess_return`
   - `max_drawdown`
   - `trade_count`
   - `annualized_turnover_rate`
   - 平均持仓周期
4. 若样本内结果不过线，则停止，不进入 OOS

**通过标准**:

- 大盘组至少出现 1-2 组可接受参数
- 平均持仓周期明显高于当前高频基线
- 年化换手率不高于 8%
- 年化超额至少不为明显负值

**停止条件**:

- 持仓周期仍然很短
- 换手率没有下降
- 样本内已普遍负超额

### 任务 D：`399006.SZ` 小范围专项治理

**目标**:

- 不再把 `399006.SZ` 混在整体模板里处理

**执行项**:

1. 仅在稳定模板附近做小扰动
2. 固定 `confirm_days/cooldown_days/min_hold_bars`
3. 只动：
   - `fast_ma`
   - `slow_ma`
   - `atr_period`
   - `atr_ma_window`
   - `buy_cap`
   - `sell_trigger`
   - `regime_filter_ma`
   - `risk_drawdown_stop`
4. 先看 `OOS`，再看 `Full`

**通过标准**:

- `OOS annualized_excess_return >= 0`
- `Full annualized_excess_return` 不再显著为负
- 换手不恶化

**停止条件**:

- 参数空间持续扩大
- 只能在 `Full` 变好，`OOS` 反而变差

### 任务 E：冻结低价值分支

**目标**:

- 停止把时间继续投入到暂时没有希望的分支上

**冻结对象**:

- `ma_convergence`
- `multi_factor_adaptive`
- `optimized_strategy_v2/v3`

**冻结条件**:

- 在没有新理论改动前，不再继续大规模调参
- 仅保留结果归档与后续对照用途
