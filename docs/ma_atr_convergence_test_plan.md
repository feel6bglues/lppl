# MA均线收敛 + ATR波动率策略测试计划

日期：2026-03-31

本文档面向第一次接触本项目的人，目标是设计并测试"布林带收口 + ATR动态分位数"作为主交易信号的策略。

这份计划的核心不是"把参数全网格扫一遍"，而是：

1. 先确认新指标计算逻辑正确
2. 再验证"低波收敛买入 + 高波收敛卖出"的策略有效性
3. 最后与现有MA+ATR基线对比，判断新策略是否有改进

如果你只想知道一句话结论：新策略尝试用**布林带收口**判断均线收敛状态，用**ATR历史分位数**判断波动率位置，目标是改善现有MA+ATR策略在震荡市的虚假信号问题。

---

## 1. 本计划覆盖什么

当前测试对象是"布林带收口 + ATR动态分位数"策略，不是现有的纯双均线+ATR策略。

当前实现位置（需要新建）：

- 策略信号与回测：`src/investment/backtest.py` (新增 `ma_convergence_atr_v1`)
- 参数调优与批量回测：`scripts/tune_ma_convergence.py` (新建)
- 结果评分与门槛：`src/investment/tuning.py` (已有，可复用)
- 当前默认参数：`config/optimal_params.yaml` (新增策略模板)

当前新策略的关键规则是：

- 买入条件：
  1. 布林带收口 (BB_width < 历史N日分位阈值)
  2. ATR处于历史低波区间 (atr_percentile < 20%)
  3. 价格突破布林上轨 或 均线向上发散
- 卖出条件：
  1. 布林带收口 (BB_width < 历史N日分位阈值)
  2. ATR处于历史高波区间 (atr_percentile > 80%)
  3. 均线死叉 或 趋势反转确认
- 风险层：
  - 长期趋势过滤 (MA120)
  - 回撤停机

当前代码里还额外存在但暂未纳入主调优网格的控制项：

- `bb_std`: 布林带标准差倍数
- `atr_percentile_window`: 分位计算窗口
- `breakout_confirmation`: 突破确认天数

## 2. 新人先看这个

如果你从来没接触过这个项目，先按这个顺序理解：

1. `src/investment/backtest.py` 负责计算布林带、ATR分位数、均线收敛度，并生成交易信号

- 输入：历史行情 + 布林带指标 + ATR历史分位数
- 中间层：收口判断、低波/高波识别、突破/反转确认
- 输出：每日仓位、交易记录、收益/回撤/换手统计

## 4. 测试目标

这轮测试要回答 4 个问题：

1. 布林带收口 + ATR低波分位 是否能有效识别"底部蓄势"状态？
2. 布林带收口 + ATR高波分位 是否能有效识别"顶部逃顶"时机？
3. 新策略是否比现有MA+ATR基线有改进？
4. 新策略对不同类型指数的适应性如何？

## 5. 通过标准

### 5.1 功能正确性标准

单元测试层面，下面这些行为必须成立：

- 布林带收口 + ATR低波 + 突破 → 买入
- 布林带收口 + ATR高波 + 死叉 → 卖出
- 非收口状态 → 不触发交易
- 风险层（趋势过滤、回撤停机）必须生效

### 5.2 结果正确性标准

回测层面，建议先看这三个主指标：

- 年化超额收益：越高越好
- 最大回撤：越小越好，注意这里是负数，数值越接近0越好
- whipsaw_rate：越低越好

次要指标：

- trade_count
- turnover_rate
- eligible

### 5.3 目标标准

本计划建议的目标分两层：

- 最低目标：不明显差于现有MA+ATR基线
- 更好目标：在保持收益的前提下，显著降低whipsaw_rate和交易次数

简单说：

- 如果信号更少了，但收益也变差了，不算通过
- 如果信号更多了，但whipsaw爆炸了，也不算通过
- 真正通过是"信号更精准 + 收益稳健"

## 9. 如何判断"达成效果"

这部分最重要，因为测试不是只看"有没有报错"。

### 9.1 如果目标是验证低波蓄势有效

你应该看到：

- 买入信号集中在ATR低波区间
- 突破前的盘整期有明显收口特征
- 突破后趋势延续性较好

### 9.2 如果目标是验证高波逃顶有效

你应该看到：

- 卖出信号在ATR高波区间触发
- 布林带在高点呈现收口特征（能量积蓄后的释放）
- 相比现有策略，顶部逃逸更快

### 9.3 如果目标是验证整体改进

你应该看到：

- 信号总数减少
- whipsaw_rate下降
- 年化超额收益不显著降低
- 至少核心宽基不退步

## 10. 当前已知风险

这份计划必须明确指出当前问题，否则新人会误判结果。

### 10.1 新策略实现复杂度

布林带收口 + ATR分位数的组合比纯双均线更复杂，可能存在：

- 参数组合过多，难以全覆盖
- 布林带和ATR的分位计算可能有滞后
- 收口判断的时间窗口选择影响较大

### 10.2 与现有策略的比较基准

当前基线是MA+ATR的Stage 4结果，新策略需要：

- 在相同时间区间对比
- 使用相同的评价指标
- 考虑策略复杂度带来的实现风险

### 10.3 ATR分位数的计算成本

ATR分位数需要计算历史窗口的排序，252天窗口的计算量较大：

- 需要优化计算效率
- 或使用近似分位数算法

## 附录：新策略核心公式

### 布林带收口判断

```
BB_middle = MA(close, bb_period)
BB_std = STD(close, bb_period)
BB_upper = BB_middle + bb_std * bb_std_mult
BB_lower = BB_middle - bb_std * bb_std_mult
BB_width = (BB_upper - BB_lower) / BB_middle
BB_width_percentile = PERCENTILE(BB_width, atr_percentile_window)
```

### ATR动态分位数

```
TR = MAX(H-L, |H-PC|, |L-PC|)
ATR = MA(TR, atr_period)
ATR_MA = MA(ATR, atr_ma_window)
ATR_ratio = ATR / ATR_MA
ATR_percentile = PERCENTILE(ATR_ratio, atr_percentile_window)
```

### 买入条件

```
买入信号 = (BB_width_percentile < bb_width_cap)
           AND (ATR_percentile < atr_low_percentile)
           AND (价格突破BB_upper OR 均线向上发散)
```

### 卖出条件

```
卖出信号 = (BB_width_percentile < bb_width_cap)
           AND (ATR_percentile > atr_high_percentile)
           AND (均线死叉 OR 趋势反转)
```
