# MA + ATR 策略测试计划

日期：2026-03-30

本文档面向第一次接触本项目的人，目标是把当前这版“纯双均线 + ATR”策略的测试方法写成一份可直接执行的说明。

这份计划的核心不是“把参数全网格扫一遍”，而是分层验证：

- 先确认代码逻辑正确
- 再确认关键参数组合行为符合预期
- 最后用 8 个指数的历史回测结果判断策略是否真的变好

如果你只想知道一句话结论：先以 `output/ma_atr_tuning_round6_w30_2012_2025_sell_atr_expand` 作为当前基线，再测试新规则是否能同时保住收益、压住回撤，并把换手/反复打脸控制住。

## 1. 本计划覆盖什么

当前测试对象是纯 `MA + ATR` 策略，不是 LPPL overlay 策略。

当前实现位置：

- 策略信号与回测：`src/investment/backtest.py`
- 参数调优与批量回测：`scripts/tune_ma_atr_only.py`
- 结果评分与门槛：`src/investment/tuning.py`
- 当前默认参数：`config/optimal_params.yaml`

当前纯策略的关键规则是：

- 买入：`MA` 金叉 + `ATR` 低波动确认
- 卖出：`MA` 死叉 + `ATR` 持续放大确认
- 长期趋势过滤：`MA120/MA180/MA240` 一类的风险过滤层
- 回撤停机：大回撤时强制降仓或清仓
- 交易抑制：`confirm_days`、`cooldown_days`、`min_hold_bars`

当前代码里还额外存在但暂未纳入主调优网格的控制项：

- `atr_deadband`
- `atr_confirm_enabled`
- `slope_threshold`

这几个参数会影响买卖触发的“硬度”，适合做补充测试，不适合一开始就混进主网格里。

## 2. 新人先看这个

如果你从来没接触过这个项目，先按这个顺序理解：

1. `src/investment/backtest.py` 负责把价格数据变成交易动作。
2. `scripts/tune_ma_atr_only.py` 负责把不同参数组合批量跑出来。
3. `src/investment/tuning.py` 负责判断“是否可接受”和“如何排序”。
4. `output/` 目录保存每次实验结果，最重要的是 `summary/*.csv` 和 `reports/*.md`。

你可以把整个系统理解成：

- 输入：历史行情
- 中间层：均线、ATR、趋势过滤、回撤过滤
- 输出：每日仓位、交易记录、收益/回撤/换手统计

## 4. 测试目标

这轮测试要回答 4 个问题：

1. 买入是否真的只在“金叉 + ATR 低波动 + 趋势允许”时发生。
2. 卖出是否真的只在“死叉 + ATR 持续放大”时发生。
3. 风险层是否能在不破坏主逻辑的前提下，压住回撤。
4. 当前这套规则是否比 `round6` 更稳，或者至少更不差。

## 5. 通过标准

### 5.1 功能正确性标准

单元测试层面，下面这些行为必须成立：

- 金叉 + ATR 合格 -> 买入
- 金叉但 ATR 不合格 -> 不买
- 死叉 + ATR 持续放大 -> 卖出或减仓
- 死叉但 ATR 不持续放大 -> 不卖
- LPPL overlay 不应干扰纯 `ma_cross_atr_v1`
- `hold` 日不应该产生额外交易
- `cooldown`、`confirm_days`、`min_hold_bars` 必须生效

### 5.2 结果正确性标准

回测层面，建议先看这三个主指标：

- 年化超额收益：越高越好
- 最大回撤：越小越好，注意这里是负数，数值越接近 0 越好
- whipsaw_rate：越低越好

次要指标：

- trade_count
- turnover_rate
- eligible

重要说明：

- 当前 `eligible` 仍被 `turnover_cap` 卡住
- 在这个问题没有修正前，不要把 `eligible=0` 直接当成策略没价值
- 先看收益、回撤、whipsaw 的方向，再看 `eligible`

### 5.3 目标标准

本计划建议的目标分两层：

- 最低目标：不低于 `round6` 的总体基线
- 更好目标：在不显著增加回撤的前提下，继续提升年化超额收益

简单说：

- 如果收益上去了，但回撤炸了，不算通过
- 如果回撤压下去了，但收益整体转负，也不算通过
- 真正通过是“收益和风险同时能接受”

## 7. 推荐测试顺序

不要一上来就跑 8 指数全量大网格。建议按下面顺序来。

### 7.1 第一轮：单元测试

先跑最小行为测试，确认没有逻辑回归。

推荐命令：

```bash
.venv/bin/python -m unittest tests.unit.test_investment_backtest
```

必须确认：

- 全部通过
- 没有因为新参数引入新的行为偏差

### 7.2 第二轮：3 指数烟雾测试

只跑这 3 个指数：

- `000300.SH`
- `000905.SH`
- `399006.SZ`

原因：

- `000300.SH`：大盘代表
- `000905.SH`：宽基中盘代表
- `399006.SZ`：高波动代表

推荐命令：

```bash
.venv/bin/python scripts/tune_ma_atr_only.py \
  --symbols 000300.SH,000905.SH,399006.SZ \
  --start-date 2012-01-01 \
  --end-date 2025-12-31 \
  --workers 30 \
  --output output/ma_atr_tuning_smoke
```

烟雾测试的通过标准：

- 不能出现明显异常值
- 结果文件必须完整生成
- Stage 2 / 3 / 4 都要有输出
- 每个指数都要能解释结果为什么好或坏

## 8. 每个指数怎么判断

下面是更具体的判定思路，便于新人直接照着看 CSV。

### 8.1 `000001.SH`

角色：上证综指，偏大盘。

看点：

- 回撤能否比当前基线更低
- 是否仍保持正超额

### 8.2 `399001.SZ`

角色：深证成指，偏成长和中盘。

看点：

- 这个指数在当前版本里收益不错，但回撤很深
- 如果新规则不能明显压回撤，就不要只看收益

### 8.3 `399006.SZ`

角色：创业板，高波动。

看点：

- 这是最容易暴露“卖出太慢”或“买入过密”的指数
- 如果它长期负超额，说明策略对高波动资产还不够稳

### 8.4 `000016.SH`

角色：上证50，偏防守。

看点：

- 这个指数通常最能检验回撤控制
- 只要它的收益和回撤同时不差，说明风险层有效

### 8.5 `000300.SH`

角色：沪深300，最重要的基准之一。

看点：

- 这是最应该优先保住的指数
- 如果它退步，通常说明主信号或卖出逻辑过强或过弱

### 8.6 `000905.SH`

角色：中证500，宽基中盘。

看点：

- 这类指数最容易在趋势和震荡之间切换
- 很适合测试 `ATR` 过滤是否合理

### 8.7 `000852.SH`

角色：中证1000，高波动、小盘。

看点：

- 容易让换手和 whipsaw 爆炸
- 如果能在它身上稳住，通常说明策略对波动更敏感

### 8.8 `932000.SH`

角色：中证2000，最激进。

看点：

- 这是高噪声测试
- 如果它大幅恶化，通常不是错误，但要说明原因

## 9. 如何判断“达成效果”

这部分最重要，因为测试不是只看“有没有报错”。

### 9.1 如果目标是验证卖出修正有效

你应该看到：

- 年化超额收益不再像 `round5` 那样被压成负数
- `whipsaw_rate` 不显著上升
- 交易次数不爆炸

### 9.2 如果目标是验证风险层有效

你应该看到：

- 最大回撤比没有风险层时更小
- `000300.SH`、`000905.SH`、`000016.SH` 这类基准指数不被打穿

### 9.3 如果目标是验证整体上线价值

你应该看到：

- 8 个指数里至少一部分保持正超额
- 核心宽基指数不退步
- 回撤比历史最好版本更稳定
- `eligible` 逐步从 0 变成非 0，或者至少接近门槛

## 10. 当前已知风险

这份计划必须明确指出当前问题，否则新人会误判结果。

### 10.1 `turnover_cap` 仍然是主要卡点

当前结果里，`eligible` 仍然全被 `turnover_cap` 拦住。

这说明至少有一个问题还没解决：

- 要么 `turnover_rate` 的尺度不对
- 要么 `turnover_cap` 的阈值不对
- 要么两者的定义不匹配

在这个问题解决前，不要用 `eligible=0` 直接下结论。

### 10.2 高波动指数会天然更难看

`399006.SZ`、`000852.SH`、`932000.SH` 的指标通常比宽基差。

这不一定是错，但要解释：

- 策略是否应该对它们单独配参
- 还是应该接受更低的目标仓位

### 10.3 单一参数不可能覆盖全部指数

当前结果已经说明：

- 大盘更偏长均线
- 宽基中盘更偏中短均线
- 高波动指数更依赖更严格的风控

所以测试计划里必须保留“按指数分组”的思路。

如果你是第一次上手，就按这个清单做：

## 13. 结论

这份计划的核心原则是：

- 先验证行为正确
- 再验证参数组合正确
- 最后验证整体统计结果正确

对当前版本来说，最重要的不是“再找一个更激进的参数”，而是：

- 保住 `round6` 已经拉回来的收益
- 把回撤往下压
- 把 `turnover_cap` 的度量问题讲清楚
