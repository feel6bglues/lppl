# MA + ATR Turnover Gap 修复总结与下一轮调优计划

日期：2026-03-30

本文档给第一次接触这个项目的人看。目标很简单：

- 先看懂这轮到底修了什么
- 再看懂修复后结果说明了什么
- 最后知道下一轮应该怎么调，不要乱扩参数

## 1. 这轮到底修了什么

上一阶段发现一个核心问题：

- `Template A` 和 `Template B` 的样本外结果都不错
- 但是全量 7 指数复核里，原来一直是 `0/7 eligible`

原因不是策略突然失效，而是换手率口径有问题。

### 1.1 原来的问题

原先的 `turnover_rate` 是：

- `总交易名义金额 / 初始资金`

这其实是**累计换手率**，不是年化换手率。

但 `turnover_cap` 的门槛，实际是按**年化口径**理解的。

结果就是：

- 5 年左右的样本外测试看起来没问题
- 14 年的全量测试会因为累计值过大被错误拒绝

### 1.2 这轮修复后怎么处理

现在的处理方式是：

- 保留 `turnover_rate`，作为累计换手率显示
- 新增 `annualized_turnover_rate`
- 用 `annualized_turnover_rate` 去和 `turnover_cap` 比较
- `turnover_gap = annualized_turnover_rate - turnover_cap`

这样 `turnover_gap` 才有可比性。

## 2. 这轮验证结果说明了什么

修复后的真实结果如下：

- `IS`：`6/7 eligible`
- `OOS`：`7/7 eligible`
- `FULL`：`6/7 eligible`

对应的均值是：

- `IS` 平均年化超额：`+2.33%`
- `OOS` 平均年化超额：`+2.93%`
- `FULL` 平均年化超额：`+2.04%`

换手方面：

- `IS` 平均年化换手：`4.02%`
- `OOS` 平均年化换手：`1.11%`
- `FULL` 平均年化换手：`5.76%`

### 2.1 结论

这说明两件事：

1. `turnover_gap` 修复是有效的，门槛不再把全量结果错误一刀切掉。
2. 策略本身仍然存在结构性分化，尤其是 `399006.SZ` 在全量里仍然是唯一的负超额失败项。

换句话说：

- 口径问题解决了
- 但策略还不是“所有指数都通吃”

## 3. 现在已经可以确定的基线

这一轮之后，后续调优不要再把所有东西混在一起。

### 3.1 可以作为稳定基线的部分

- `Template A` 在 OOS 上稳定
- `Template B` 在 OOS 上也稳定
- `confirm_days = 2`
- `cooldown_days = 5`
- `min_hold_bars = 3`

这三个交易节奏参数已经基本收敛，不值得再大幅折腾。

### 3.2 需要继续关注的部分

- `399006.SZ` 全量仍然负超额
- 全量回撤还是偏高
- `turnover_cap` 未来是否需要模板分层，或者指数分层

## 4. 下一轮调优目标

下一轮不要再做大网格，也不要再重新引入 `atr_deadband / slope_threshold / atr_confirm_enabled` 的专项实验。

下一轮只做三件事：

1. 继续核实 `turnover_gap` 和 `turnover_cap` 是否真的同口径
2. 针对 `399006.SZ` 单独做小范围参数收缩
3. 检查是否需要把 `turnover_cap` 拆成模板级门槛，而不是一刀切

## 5. 下一轮怎么做

### 5.1 先看 `399006.SZ`

`399006.SZ` 是当前唯一明显不稳定的指数。

它的问题不是换手，而是：

- 全量年化超额为负
- 回撤相对也不轻

所以它应该单独处理，不要和其他指数共用完全一样的参数模板。

### 5.2 推荐测试方向

对 `399006.SZ`，下一轮只测小范围组合：

- `fast_ma`: `5, 10, 20`
- `slow_ma`: `30, 60, 120`
- `atr_period`: `10, 14, 20`
- `atr_ma_window`: `20, 40`
- `buy_cap`: `1.00, 1.05, 1.10`
- `sell_trigger`: `1.00, 1.05`
- `regime_filter_ma`: `120, 180, 240`
- `risk_drawdown_stop`: `0.12, 0.15, 0.18`

原则是：

- 不扩展到更大网格
- 只在现有稳定模板附近微调
- 优先看样本外，不要只看全量

### 5.3 对其他指数

对已经稳定的指数，不要再大动。

重点是：

- 保持 OOS 稳定
- 保持年化换手不过高
- 确认全量和样本外的口径一致

## 6. 通过标准

下一轮如果满足下面条件，就可以认为调优有进展：

- `turnover_gap` 的计算方式和报告展示完全一致
- `FULL` 不再因为口径错误被错误拒绝
- `399006.SZ` 的年化超额从负值拉回到非负
- OOS 结果不比当前版本明显变差
- 不靠扩大参数空间来“刷出”结果

## 7. 失败标准

如果出现下面情况，就说明下一轮没调对：

- 又开始扩 `MA / ATR` 大网格
- 又重新把 `atr_deadband / slope_threshold / atr_confirm_enabled` 混回主实验
- 只看全量，不看 OOS
- 结果里 `turnover_rate`、`annualized_turnover_rate`、`turnover_gap` 三者口径不一致

## 8. 建议的下一轮输出目录

建议单独开一组目录，避免和这轮混淆：

- `output/ma_atr_turnover_gap_next/is`
- `output/ma_atr_turnover_gap_next/oos`
- `output/ma_atr_turnover_gap_next/full`

## 9. 最后一句话

这轮已经把“换手口径错了”这个问题修掉了。

下一轮不要再解决同一个问题，而是要回答：

- 在口径统一之后，`399006.SZ` 还能不能被单独调到可用
- 如果不能，是否要给它单独模板，或者直接降级处理
