# LPPL 系统详细使用文档（v1.3.1）

本文档基于当前代码实现整理，覆盖环境准备、数据配置、主流程运行、验证流程、输出解读与常见问题排查。

## 1. 系统简介

LPPL（Log-Periodic Power Law）系统用于识别金融市场潜在泡沫风险，当前版本已完成以下关键能力：

- Numba JIT 加速核心计算
- joblib 并发执行计算任务
- 本地通达信（TDX）数据优先读取
- akshare 兜底读取中证2000（`932000.SH`）
- 单窗口与 Ensemble（多窗口共识）验证模式
- Walk-Forward 盲测验证链路

## 2. 目录与核心入口

项目主目录关键文件：

- `main.py`：主扫描兼容入口（wrapper）
- `lppl_verify_v2.py`：历史高点验证兼容入口（wrapper）
- `lppl_walk_forward.py`：Walk-Forward 盲测兼容入口（wrapper）
- `src/cli/main.py`：主扫描真实实现
- `src/cli/lppl_verify_v2.py`：验证真实实现
- `src/cli/lppl_walk_forward.py`：Walk-Forward 真实实现
- `src/cli/generate_optimal8_report.py`：8指数可读报告真实实现
- `scripts/`：研究/实验脚本目录（网格搜索、回测、历史实验脚本）
- `docs/archive/`：历史规划文档归档
- `src/data/manager.py`：数据管理与数据源优先级控制
- `src/data/tdx_reader.py`：TDX `.day` 本地二进制读取器
- `src/constants.py`：指数、目录、开关等配置常量
- `output/`：默认输出目录

## 3. 环境准备

### 3.1 Python 与虚拟环境

建议 Python 3.12：

```bash
python -m venv .venv
source .venv/bin/activate
```

### 3.2 安装依赖

```bash
pip install -r requirements.txt
pip install 'mootdx[all]'
```

说明：

- `numba`：LPPL 核心函数 JIT 加速
- `joblib`：并发执行
- `mootdx/tdxpy`：读取本地通达信数据
- `akshare`：远程补充 `932000.SH`

## 4. 数据源与读取优先级

系统支持 8 个指数：

- 本地 TDX 优先（7个）
  - `000001.SH`、`399001.SZ`、`399006.SZ`
  - `000016.SH`、`000300.SH`、`000905.SH`、`000852.SH`
- akshare 远程（1个）
  - `932000.SH`

读取优先级（由 `DataManager` 控制）：

1. 本地指数先读 TDX `.day`
2. `932000.SH` 走 akshare
3. 失败时回退到本地 parquet 缓存

## 5. 环境变量配置（可选）

如不设置，系统使用代码中的默认目录。

```bash
# 通达信 vipdoc 根目录
export LPPL_TDX_DATA_DIR="/home/james/.local/share/tdxcfv/drive_c/tc/vipdoc"

# 验证输出目录总开关
export LPPL_VERIFY_OUTPUT_DIR="output/MA"

# 子目录（可按需覆盖）
export LPPL_PLOTS_DIR="output/MA/plots"
export LPPL_REPORTS_DIR="output/MA/reports"
export LPPL_SUMMARY_DIR="output/MA/summary"
export LPPL_RAW_DIR="output/MA/raw"
```

### 5.1 快速上手（3条命令）

```bash
# 1) 安装依赖
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2) 先跑验证（推荐：按指数最优参数）
.venv/bin/python lppl_verify_v2.py --all --ensemble --use-optimal-config

# 3) 再跑盲测（单指数示例）
.venv/bin/python lppl_walk_forward.py --symbol 000001.SH --ensemble --use-optimal-config
```

说明：
- 若不启用最优参数模式，删除 `--use-optimal-config` 即可。
- 最优参数默认读取 `config/optimal_params.yaml`，可用 `--optimal-config-path` 改路径。

## 6. 主程序使用（批量扫描）

### 6.1 运行命令

```bash
.venv/bin/python main.py
```

### 6.2 执行流程

`main.py` 分三步执行：

1. 数据可用性检查并加载指数数据
2. LPPL 并发计算与结果汇总
3. 生成 HTML 可视化报告

### 6.3 产出文件

默认在 `output/` 下生成：

- `lppl_report_YYYYMMDD.md`
- `lppl_report_YYYYMMDD.html`
- `lppl_params_YYYYMMDD.json`

`YYYYMMDD` 优先采用输入数据里的最新交易日。

## 7. 验证程序使用（lppl_verify_v2.py）

### 7.1 常用命令

```bash
# 默认验证上证综指
.venv/bin/python lppl_verify_v2.py

# 验证单个指数
.venv/bin/python lppl_verify_v2.py --symbol 000001.SH

# 验证所有指数
.venv/bin/python lppl_verify_v2.py --all

# Ensemble 模式
.venv/bin/python lppl_verify_v2.py --symbol 000001.SH --ensemble

# 自定义参数
.venv/bin/python lppl_verify_v2.py --all --ensemble --max-peaks 5 --step 5 --ma 5

# 启用按指数最优参数（YAML）
.venv/bin/python lppl_verify_v2.py --all --ensemble --use-optimal-config
```

### 7.2 参数说明

- `--symbol, -s`：指数代码（如 `000001.SH`）
- `--all, -a`：验证全部 8 个指数
- `--ensemble, -e`：启用多窗口共识模式
- `--max-peaks, -m`：每个指数最多分析的高点数（默认 10）
- `--step`：扫描步长（默认 5）
- `--ma`：移动平均窗口（默认 5）
- `--output, -o`：输出根目录（默认 `output/MA`）
- `--use-optimal-config`：启用按指数最优参数模式（从 YAML 读取）
- `--optimal-config-path`：最优参数 YAML 路径（默认 `config/optimal_params.yaml`）

### 7.3 两种模式差异

- 单窗口模式：窗口 `40/60/80`（3个窗口）
- Ensemble 模式：`40~150`，步长 `10`（12个窗口）

Ensemble 模式默认共识阈值为 25%，即 12 个窗口中至少 3 个窗口达成共识才触发信号。

### 7.4 最优参数模式说明

- 默认配置文件：`config/optimal_params.yaml`
- 文件结构：
  - `window_sets`：窗口集合定义
  - `symbols`：各指数最优参数
  - `defaults`：全局默认参数
- 生效逻辑：
  - 开启 `--use-optimal-config` 后，按 `symbol` 读取并覆盖运行参数
  - 会在终端打印每个指数“生效参数”和 `source`
- 回退策略：
  - 配置文件缺失/解析失败，或某指数缺失配置时，自动回退默认参数并打印告警，不中断任务

### 7.5 验证输出

输出目录结构（默认 `output/MA`）：

- `raw/`：逐案例 timeline 原始 parquet
- `plots/`：案例图、共识图、统计图
- `reports/`：Markdown 与 HTML 验证报告
- `summary/`：CSV 汇总

典型文件示例：

- `summary/peak_verification_v2_single_window.csv`
- `summary/peak_verification_v2_ensemble.csv`
- `reports/verification_report_v2_single_window.md`
- `reports/verification_report_v2_ensemble.html`

## 8. Walk-Forward 盲测使用

### 8.1 常用命令

```bash
# 单窗口模式
.venv/bin/python lppl_walk_forward.py --symbol 000001.SH

# Ensemble 模式
.venv/bin/python lppl_walk_forward.py --symbol 000001.SH --ensemble

# 自定义参数
.venv/bin/python lppl_walk_forward.py --symbol 000001.SH --step 5 --lookahead 60 --drop-threshold 0.10

# 启用按指数最优参数（YAML）
.venv/bin/python lppl_walk_forward.py --symbol 000001.SH --ensemble --use-optimal-config
```

### 8.2 参数说明

- `--symbol, -s`：指数代码，默认 `000001.SH`
- `--ensemble, -e`：启用 Ensemble
- `--step`：扫描步长，默认 5
- `--lookahead`：未来观察天数，默认 60
- `--drop-threshold`：未来跌幅阈值，默认 0.10
- `--output, -o`：输出目录，默认 `output/MA/summary`
- `--use-optimal-config`：启用按指数最优参数模式（从 YAML 读取）
- `--optimal-config-path`：最优参数 YAML 路径（默认 `config/optimal_params.yaml`）

### 8.3 输出文件

- `walk_forward_<SYMBOL>_single_window.csv`
- `walk_forward_<SYMBOL>_single_window_summary.csv`
- `walk_forward_<SYMBOL>_ensemble.csv`
- `walk_forward_<SYMBOL>_ensemble_summary.csv`

### 8.4 8指数可读报告（固化模板）

当你已经产出 8 指数汇总（`walk_forward_optimal_8index_summary_*.csv`）后，可直接生成高可读报告：

```bash
.venv/bin/python generate_optimal8_report.py \
  --summary-csv output/MA/summary/walk_forward_optimal_8index_summary_YYYYMMDD_HHMMSS.csv
```

输出文件：
- `output/MA/reports/optimal8_human_friendly_report_v2_*.md`
- `output/MA/plots/optimal8_*_readable_*.png`

## 9. 风险等级规则

当前系统对预警天数（`days_left`）采用如下分级：

- `< 5`：极高危（DANGER）
- `< 20`：高危（Warning）
- `< 60`：观察（Watch）
- `>= 60`：安全（Safe）
- 参数无效：无效模型（假信号）

## 10. 质量门禁与验证流程

建议在改动后执行：

```bash
# 1) 编译检查
.venv/bin/python -m compileall src main.py lppl_verify_v2.py lppl_walk_forward.py

```

## 11. 常见问题排查

### 11.1 报错 `TDX directory not found`

- 检查 `LPPL_TDX_DATA_DIR` 是否指向 `vipdoc`
- 确认目录下存在 `sh/lday`、`sz/lday`

### 11.2 `932000.SH` 拉取失败

- 检查网络与 akshare 版本
- 可先使用已有 parquet 缓存继续分析

### 11.3 输出目录文件不全

- 先看终端日志是否有“无数据”“分析失败”提示
- 检查 `--output` 与环境变量目录是否一致

### 11.4 速度异常慢

- 首次运行包含 Numba 编译开销，二次运行通常会更快
- 验证 CPU 核心可用性与并发设置

## 12. 推荐日常执行顺序

1. 更新/检查数据源（TDX + 缓存）
2. 跑 `main.py` 生成当日扫描报告
3. 对重点指数运行 `lppl_verify_v2.py --ensemble`
4. 定期执行 `lppl_walk_forward.py` 做盲测回归
5. 代码变更后执行完整质量门禁
